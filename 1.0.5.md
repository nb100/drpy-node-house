# V1.0.5 功能计划与详细设计文档

## 1. 概述
本版本主要引入用户激励体系，包括积分系统、头衔（段位）系统以及基于积分的内容交互功能（悬赏、付费查看）。

## 2. 积分系统 (Points System)

### 2.1 积分获取规则
- **每日签到**: +10 积分（每日一次）。
- **发布话题**: +5 积分（每日上限 5 次，防止灌水）。
- **回复话题**: +2 积分（每日上限 20 次）。
- **话题加精**: +50 积分（管理员操作，额外奖励）。
- **被采纳回答**: 获得悬赏积分。
- **内容收益**: 发布的付费话题被购买时获得积分。

### 2.2 积分消耗规则
- **发布悬赏**: 扣除对应悬赏积分（发帖时冻结/扣除）。
- **查看付费内容**: 扣除设定积分。

## 3. 头衔/段位系统 (Rank System)

## 4. 数据库设计 (Schema Changes)

### 4.1 Users 表变更
新增字段：
- `points`: INTEGER, Default 0 (当前积分)
- `last_checkin_date`: TEXT (YYYY-MM-DD，用于判断今日是否签到)

### 4.2 Topics 表变更
新增字段：
- `view_permission_level`: INTEGER, Default 0 (观看所需最低段位等级，0为不限)
- `view_points_required`: INTEGER, Default 0 (观看所需积分，0为免费)
- `bounty_points`: INTEGER, Default 0 (悬赏积分，>0 即为悬赏帖)
- `is_solved`: INTEGER, Default 0 (是否已结帖/已解决)
- `solved_comment_id`: INTEGER (被采纳的评论ID)

### 4.3 新增表：UserPointsHistory (积分记录)
用于记录每一笔积分变动，方便查询和回溯。
```sql
CREATE TABLE user_points_history (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    amount INTEGER NOT NULL, -- 正数为增加，负数为消耗
    reason TEXT NOT NULL, -- 类型: checkin, topic_post, topic_reply, bounty_post, bounty_reward, view_pay, view_earn, admin_adjust
    related_id INTEGER, -- 关联的 topic_id 或 comment_id
    created_at INTEGER DEFAULT (strftime('%s', 'now')),
    FOREIGN KEY(user_id) REFERENCES users(id)
);
```

### 4.4 新增表：TopicPurchases (话题购买记录)
记录用户购买了哪些付费话题的权限。
```sql
CREATE TABLE topic_purchases (
    user_id INTEGER NOT NULL,
    topic_id INTEGER NOT NULL,
    amount INTEGER NOT NULL,
    created_at INTEGER DEFAULT (strftime('%s', 'now')),
    PRIMARY KEY (user_id, topic_id),
    FOREIGN KEY(user_id) REFERENCES users(id),
    FOREIGN KEY(topic_id) REFERENCES topics(id)
);
```

## 5. 接口设计 (API Endpoints)

### 5.1 用户相关
- `POST /api/user/checkin`: 执行每日签到。
- `GET /api/user/points/history`: 获取积分变动记录。
- `GET /api/user/rank`: 获取当前段位信息（或包含在用户信息中）。

### 5.2 论坛相关
- `POST /api/topics` (修改): 支持传入 `bounty_points`, `view_points_required`, `view_permission_level`。
- `POST /api/topics/:id/solve`: 采纳评论（仅楼主或管理员可用）。
    - 参数: `comment_id`
    - 逻辑: 将悬赏积分转给评论作者，标记话题为已解决。
- `POST /api/topics/:id/purchase`: 购买付费话题查看权限。
- `GET /api/topics/:id` (修改):
    - 如果是付费/限制话题，且用户未购买/等级不足，仅返回标题和摘要，内容字段隐藏或打码，并返回 `access_denied: true` 及购买所需条件。

## 6. 前端页面与交互

### 6.1 个人中心
- 展示当前积分、段位图标/名称。
- “每日签到”按钮（已签到则禁用或显示“已签到”）。
- 积分记录列表页面。

### 6.2 论坛列表
- 悬赏帖显示“悬赏”图标及金额。
- 付费/门槛帖显示“锁”图标。
- 已解决悬赏帖显示“已解决”状态。

### 6.3 帖子详情页
- **悬赏帖**:
    - 楼主视角: 评论区显示“采纳为答案”按钮。
    - 访客视角: 显示悬赏金额。
- **付费/门槛帖**:
    - 无权限时: 显示“内容被隐藏”，提示“需要 X 积分购买”或“需要 X 段位可见”。
    - 购买按钮: 点击后弹出确认框，扣除积分并刷新显示内容。

### 6.4 发帖页面
- 新增“高级设置”区域：
    - 悬赏积分输入框（输入即开启悬赏）。
    - 阅读权限设置（下拉选择最低段位）。
    - 阅读消耗设置（输入积分数）。

## 7. 升级经验算法 (Leveling Algorithm)
简单起见，段位直接由当前积分决定，不设独立“经验值”。
- 优点: 鼓励持续活跃，积分消费后段位可能下降，促进用户赚取积分。
- 缺点: 长期不活跃用户积分不减，段位保留。
- *可选优化*: 引入“活跃度”或“赛季”概念（暂不实现，保持简单）。
